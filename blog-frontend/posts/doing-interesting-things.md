我一直在思考"有意思"这个词。

它不是宏大的，不是改变世界的，就是让你在做的时候想继续做下去的事情。那种感觉——你忘记时间，忘记外界的评价，只是在做，做完还想做。

## 什么是有意思

有趣的事情有一个共同点：它们在开始的时候不确定结果。

你不知道这段代码最终会不会跑通，不知道这篇文章会不会有人读，不知道这个项目会不会成功。但你继续做，因为过程本身就有回报。

> 有意思的事情的本质，是不断遭遇惊喜。

和功利心驱动的事情相比，有意思的事往往没有清晰的终点。没有 deadline，没有 KPI，只有你和你正在做的事情之间的关系。

## 怎么找到它

找到有意思的事情，通常不是规划出来的，是撞出来的。

有几个信号值得注意：

1. **你会主动搜索相关内容** — 不是因为工作需要，而是睡前刷着刷着就刷到了
2. **你会忘记时间** — 不是发呆，是真的在做
3. **你愿意接受失败** — 失败了还想再试一次

```python
def is_interesting(thing):
    return (
        thing.you_initiate and
        thing.you_lose_track_of_time and
        thing.failure_doesnt_stop_you
    )
```

这不是什么算法，但大致准确。

## 关于"有没有用"

很多人在开始一件事之前，会先问：这有什么用？

这个问题会杀死很多有意思的事情。

有没有用，往往只能事后判断。你在 2008 年学 iOS 开发，有没有用？在 2012 年学机器学习，有没有用？在 2020 年研究 LLM，有没有用？

现在看来都有用，但当时没人能确定。

> 停止追问"这有什么用"，是做有意思的事情的第一步。

## 一个实践建议

每周留出一点时间，做一件纯粹因为觉得好玩而做的事情。

可以是写一个没人会用的小工具，可以是学一门不知道有没有用的语言，可以是写一篇没有主题的文章。

做就对了。
